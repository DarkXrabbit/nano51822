Detailed Description

BLE Device Class (SoftDevice).

This class manages the all BLE event from the SoftDevice driver, and provides the device-manager, advertising and gap objects.

Examples:
/ble/ble_app_hrm_htm_bat/src/main.cpp, /ble/ble_app_uart/src/main.cpp, /ble/ble_beacon/src/main.cpp, /ble/ble_rssi_measure/src/main.cpp, and /peripherals/i2c_scanner/src/main.cpp.

Function for initializing the Device Information Service.

This call allows the application to initialize the device information service. It adds the DIS service and DIS characteristics to the database, using the initial values supplied through the p_dis_init parameter. Characteristics which are not to be added, shall be set to NULL in p_dis_init.

Parameters
[in]manufact_nameManufacturer Name String.
[in]model_numberModel Number String.
[in]serial_numberSerial Number String.
[in]hw_revHardware Reversion String.
[in]fw_revFirmware Reversion String.
[in]sw_revSoftware Reversion String.
[in]p_sys_idPoint to the ble_dis_sys_id_t.

NRF_SUCCESS on successful initialization of service.

virtual void CThread::run ( )
protectedvirtualinherited

The run member function is the task main procedure, and callback by RTOS when task start.

Example:
class CLedTask: public CThread {
protected:
virtual void run() {
CPin led(LED2);
while( isAlive() ) { // check thread alive
led = !led;
sleep(200);
}
}
};
int main() {
...
CLedTask ledTask;
ledTask.start("led"); // default stack=128, default priority=low
...
}
Remarks
The run() is a pure virtual function, must implement by inheritor.
Note
if end the run() member function, the CThread object will be destroyed and collected.

Reimplemented in CDebug, and CShell.

Examples:
/rtos/blink/src/main.cpp.
